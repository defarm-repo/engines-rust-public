# Documentation Organization

This directory contains all project documentation organized by category.

## Directory Structure

### `/deployment/`
Production deployment guides and configuration:
- Production deployment procedures
- Railway deployment guides
- Database setup instructions
- Infrastructure configuration

### `/development/`
Development guides and workflows:
- Testing guides
- Integration quickstarts
- Development setup
- Client demo setup

### `/api/`
API specifications and authentication:
- API structure documentation
- Authentication guides
- Backend specifications

### `/archived/`
Historical documents and resolved issues:
- Old status reports
- Resolved bug reports
- Session summaries
- Deprecated guides

## Root-Level Documentation

Only essential, frequently-accessed documents should remain in the project root:
- `README.md` - Project overview and quick start
- `CLAUDE.md` - AI assistant instructions
- `CONTRIBUTING.md` - Contribution guidelines (if exists)
- `LICENSE` - License information (if exists)
- `CHANGELOG.md` - Version history (if exists)

All other documentation should be organized into the appropriate subdirectory above.
